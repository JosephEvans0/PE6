Code Reviews
------------

1. One issue = one PR

2. minimum _______ approvals before a merge

3. assign _______ times as many reviewers

4. use blame to pick reviewers

5. bring in specialists for small sections

6. pick 1 - 2 days a week to be "review days"

7. move comments into code

8. make a merge policy as a team

9. include screenshots/gifs for UI/UX changes

10.keep it concise
